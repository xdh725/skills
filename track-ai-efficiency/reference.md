# 参考：记录示例与时间解析

## 时间解析规则

日报汇总时，将「节省时间」列解析为分钟数，支持以下表述：

| 表述 | 换算为分钟 |
|------|------------|
| 30 分钟、30min、30m | 30 |
| 半小时 | 30 |
| 1 小时、1h、1 个小时 | 60 |
| 约 1 小时、大概 1 小时 | 60 |
| 1.5 小时、1小时30分 | 90 |
| 2 小时、2h | 120 |
| 约 2 小时 5 分钟 | 125 |
| 45 分钟 | 45 |

**汇总输出**：将总分钟数转换为「X 小时 Y 分钟」格式，如 125 分钟 →「约 2 小时 5 分钟」。

无法解析时保留原文，汇总中注明「部分时间无法解析」。

## 记录示例

**单任务**：
- 任务：重构 UserDetail 模块
- 时间：约 45 分钟
- 输出行：`| 2025-03-02 | 重构 UserDetail 模块 | 约 45 分钟 |`

**多任务一次记录**：用户可一次提供多条，逐行追加：
- 任务1：修复 Feed API 分页 bug，20 分钟
- 任务2：编写单元测试，1 小时
- 输出：两行表格行

## 日报示例

```markdown
# AI 开发效率日报 - 2025-03-02

## 今日汇总
- **完成任务数**：3 项
- **预估节省时间**：约 2 小时 5 分钟

## 任务明细
| 任务 | 节省时间 |
|------|----------|
| 重构 UserDetail 模块 | 约 45 分钟 |
| 修复 Feed API 分页 bug | 约 20 分钟 |
| 编写单元测试 | 约 1 小时 |
```

## Prompt 使用记录（自动记录）

项目规则会要求 AI 在**每次回复后**自动往 `~/.cursor/ai-prompt-usage-log.md` 追加一行：

- **日期时间**：记录时刻
- **Prompt 摘要**：本轮用户问题的一句话概括
- **代码量**：本轮若有代码编辑则由 AI 估算（如「约 50 行」「2 文件」），未改代码为 `—`
